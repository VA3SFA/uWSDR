On many platforms there are often more one way in which to access the sound hardware. The reason for this is that the evolution of the use of multi-media applications requiring better (i.e. faster) access to the underlying hardware. In all cases the older methods are still available, and all of these are available for use with the uWSDR GUI.

In all lists the oldest and slowest options are listed first. The requirements made by uWSDR on the audio system are not too great so even using the old access methods is not normally a problem.

Windows

The choices available are:

  • MME
  • Windows DirectSound
  • ASIO

ASIO is not an official Microsoft standard, but is provided by independant vendors. It is supposed to be the best API to use, but both MME and Windows DirectSound are perfectly adequate.

Linux

The choices are:

  • Jack
  • OSS
  • ALSA

ALSA is the new standard and OSS is in fact a layer on top of ALSA to provide compatibility for old programs and therefore there is no advantage to using OSS if ALSA is available. Jack is a generalised method to connect a program to sound devices and/or other programs allowing for great flexibility.

Mac OS X

The choices are:

  • Jack
  • CoreAudio

CoreAudio is the standard Mac OS X method for accessing sound devices. Jack is described in the Linux section above.
